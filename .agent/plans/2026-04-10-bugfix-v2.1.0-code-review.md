## Task 4: Fix fragile script-wrapping heuristic in `apply_fix_script`

**Root cause:** `scriptContent.includes('return')` can match the word "return" in a comment or string, causing the IIFE wrapper to be skipped. The mutation silently succeeds but the return value is `undefined`, so `candidateMapped` falls back to the pre-mutation `sandbox.row` — the change is discarded without error.

**Fix:** Drop the heuristic. Always wrap the script in an IIFE. Scripts are already expected to mutate `row` directly. The IIFE wrapper just needs to return `row` at the end.

**Files:**
- Modify: `src/app/api/mcp/route.ts:131`
- Modify: `tests/app/api/mcp/route.test.ts` (add regression test)

- [ ] **Step 1: Replace the heuristic with an unconditional IIFE wrapper**

In `src/app/api/mcp/route.ts`, replace lines 130–132:

```typescript
// Before
// Wrap code in an anonymous function if it doesn't return row
const codeToRun = scriptContent.includes('return') ? scriptContent : `(function(row){ ${scriptContent}\n return row; })(row)`
const result = vm.runInNewContext(codeToRun, sandbox, { timeout: 1000 })

// After
// Always wrap in IIFE so scripts can mutate row without needing an explicit return
const codeToRun = `(function(row){ ${scriptContent}\n return row; })(row)`
const result = vm.runInNewContext(codeToRun, sandbox, { timeout: 1000 })
```

- [ ] **Step 2: Add a regression test that has "return" in a comment**

In `tests/app/api/mcp/route.test.ts`, add this test inside the `Self-Learning Tools` describe block:

```typescript

  const fs = (await import('fs/promises')).default
  // Script has "return" only in a comment, not as a statement

  const mockState = {
    batchId: 'b_comment', tenantId: 'tenant_a', status: 'MAPPED' as const,

  vi.mocked(verifyToken).mockResolvedValue({
    token: 'tok', clientId: 'c', scopes: ['mcp:tools'], extra: { tenantId: 'tenant_a' },
```
