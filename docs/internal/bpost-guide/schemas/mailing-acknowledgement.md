> **When to use this file:** When parsing the acknowledgement file returned by bpost after submitting a MailingRequest, to confirm the file was received and identify it by original file name and timestamp.

# Mailing Acknowledgement File

The Acknowledgement file confirms that the system has received a file. It is generated by bpost. It indicates the original Request file name and the time when the Request file was received. The root tag of the mailing Acknowledgement file is `<RequestAck>`.

The Acknowledgement file structure is a generic file structure that is identical for all Request files (both Deposit and Mailing).

See [mailing-request.md](mailing-request.md) for the request file this acknowledges.
See [mailing-response.md](mailing-response.md) for the full response with processing results.

**Remark:** With the Address File Tool, there is no Acknowledgement file. It is the presence of the Request file in the files list on the AFT part of the e-MassPost website that acknowledges the good reception of the Request file.

## File Structure Diagram

```
RequestAck ──────► FileReceived
(Root tag for       (Tag that contains the

```

## XML Structure

| Tag Name | Attributes | Description | Rule | Mandatory | Type | Max Length | Default |
|----------|-----------|-------------|------|-----------|------|------------|---------|
| RequestAck | | | | Yes | | | |
| RequestAck/FileReceived | fileName | Name of the received file | See [file-naming.md](file-naming.md) (File Naming Convention) | Yes | String | 50 | |
| | timeStamp | | Format is: YYYY-MM-DDThh:mm:ss e.g. 2001-12-17T09:30:47 | Yes | Timestamp | 19 | |

## TXT Structure

```
FileReceived|fileName|timeStamp
```
