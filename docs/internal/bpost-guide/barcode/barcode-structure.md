# MAIL ID Barcode Structure

## General Barcode Information

The barcode used by bpost for the MAIL ID system is a **Code 128** barcode. Code EAN 128 is **not** supported.

Code 128 is a high-density symbology that permits encoding of alphanumeric data. Key properties:

- Includes a checksum digit for verification
- Can be verified character-by-character by verifying parity of each data byte
- Encodes a large amount of data in a small space
- Allows numeric data to be encoded at double-density (using character set C)

See [code128-encoding.md](code128-encoding.md) for the full Code 128 encoding algorithm and reference table.

## Barcode Structure Breakdown (Table 8)

The MAIL ID barcode is structured as follows:

| Field | Content | Section |
|---|---|---|
| Start character | `[startcode]` | G |
| License Plate Identifier (A) | `J` | A |
| UPU Header (A) | `JBEA` | A |
| FCC (B) | e.g. `12` | B |
| Customer Identification (C) | e.g. `12345` | C |
| Mail Piece Number (D) | e.g. `12345678901` | D |
| Checksum | `[checkdigit]` | F |
| Stop character | `[endcode]` | G |

### License Plate Identifier (A)

Indicates that the issuing agency is a Universal Postal Union (UPU) member. This field is fixed and always contains the letter **J** (capital letter required).

### UPU Header (A)

Indicates the postal organization for which the barcode is used. This field is fixed and always contains **JBEA** (capital letters required).

**Subset switch (optional):** A Code_C (Code 128 value 99) can optionally be entered between the UPU Header field (A) and the FCC field (B). This switches to subset C for all numeric characters that follow, producing the most compact barcode.

## The MAIL ID Number (B + C + D)

The MAIL ID number consists of three consecutive fields:

1. **Format Control Code (FCC)** -- field B
2. **Customer Identification (BarcodeCustomerId)** -- field C
3. **Mail Piece Number** -- field D

**The MAIL ID number must be unique over a period of at least 30 days.**

### Format Control Code (FCC) -- Field B

The FCC identifies the barcode format (the configuration of fields and bars). It is a 2-digit numeric code:

- **First digit:** `1` if the customer generates the barcode, `9` if generated by bpost
- **Second digit:** indicates the number of digits in the mail piece number

| FCC Value | Mail Number Digits | Generated By |
|---|---|---|
| 10 | 7 digits | Customer |
| 90 | 7 digits | bpost |
| 11 | 9 digits | Customer |
| 91 | 9 digits | bpost |
| 12 | 11 digits | Customer |
| 92 | 11 digits | bpost |

**Example:**
- `121234507256000001` -- generated by the customer (FCC starts with `1`)
- `921234507256000001` -- generated by bpost (FCC starts with `9`)

### Customer Identification (BarcodeCustomerId) -- Field C

The CustomerBarcodeID identifies the party responsible for creating the barcodes and mail pieces. It is a **5-digit code** (left-padded with zeros if necessary) assigned by bpost to each customer who enters the MAIL ID program.

### Mail Piece Number -- Field D

The mail piece number is the core of the barcode. The customer is free to organize the mail number as desired. Ranges within the mail number can be reserved for applications.

## Mail Number Uniqueness Requirement

The MAIL ID number (FCC + BarcodeCustomerId + Mail Piece Number) **must be unique for a minimum of 30 days**. This ensures that bpost's sorting systems can unambiguously identify each mail piece during processing.
